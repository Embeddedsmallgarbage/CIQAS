# 设置界面功能 Spec

## Why
当前知识库管理按钮仅对管理员可见，功能单一。需要将其改为设置按钮，整合知识库管理和学生账号管理功能，方便管理员统一管理系统设置。

## What Changes
- 将侧边栏底部的「知识库管理」按钮改为「设置」按钮
- 创建新的设置模态框，包含两个设置板块
- 新增学生账号管理功能，支持添加学生账号
- 设置按钮对所有登录用户可见，但学生只能查看部分设置

## Impact
- Affected code: templates/index.html, static/css/style.css, static/js/main.js, app.py, auth.py
- New API: 学生账号增删改查接口

## ADDED Requirements

### Requirement: 设置按钮
系统应在侧边栏底部提供设置按钮，所有登录用户可见。

#### Scenario: 管理员查看设置
- **WHEN** 管理员登录后查看侧边栏
- **THEN** 显示设置按钮
- **AND** 点击后打开设置界面，显示所有设置板块

#### Scenario: 学生查看设置
- **WHEN** 学生登录后查看侧边栏
- **THEN** 显示设置按钮
- **AND** 点击后打开设置界面，仅显示部分设置（不显示知识库管理和学生账号管理）

### Requirement: 设置界面布局
设置界面应采用标签页或侧边栏导航布局。

#### Scenario: 设置界面结构
- **WHEN** 用户打开设置界面
- **THEN** 显示左侧导航栏和右侧内容区
- **AND** 左侧导航栏显示可用的设置分类
- **AND** 右侧显示当前选中分类的内容

#### Scenario: 管理员设置分类
- **WHEN** 管理员打开设置界面
- **THEN** 左侧导航栏显示「知识库管理」和「学生账号管理」两个分类

### Requirement: 知识库管理设置
保留原有知识库管理功能，迁移到设置界面中。

#### Scenario: 知识库管理功能
- **WHEN** 管理员选择「知识库管理」分类
- **THEN** 显示文档上传和文档列表功能
- **AND** 功能与原有知识库管理一致

### Requirement: 学生账号管理
管理员可以添加、查看、删除学生账号。

#### Scenario: 查看学生列表
- **WHEN** 管理员选择「学生账号管理」分类
- **THEN** 显示已添加的学生账号列表
- **AND** 显示每个学生的用户名和名称

#### Scenario: 添加学生账号
- **WHEN** 管理员点击「添加学生」按钮
- **THEN** 显示添加表单，包含用户名、密码、名称字段
- **AND** 提交后创建学生账号
- **AND** 密码使用哈希存储

#### Scenario: 删除学生账号
- **WHEN** 管理员点击学生账号的删除按钮
- **THEN** 显示确认对话框
- **AND** 确认后删除该学生账号

#### Scenario: 学生账号登录
- **WHEN** 使用新添加的学生账号登录
- **THEN** 登录成功，可以访问问答系统
- **AND** 无法访问知识库管理和学生账号管理

### Requirement: 设置界面 UI/UX
设置界面应遵循专业 UI 设计规范。

#### Scenario: 设计规范
- **WHEN** 渲染设置界面
- **THEN** 使用与主系统一致的配色方案
- **AND** 使用 SVG 图标
- **AND** 所有按钮最小高度 44px
- **AND** 表单有正确的 label 关联
